Contributing to the Meta-UI Kit

We think it would be great for you to help improve the Meta-UI Kit by forking and making beneficial changes. Please adhere to the guidelines below to make the best out of your contributions. It will also make it easier for our developers to determine if your changes are to be added to our kit.

Developer Setup Instructions

#1. First, clone our repository from Github. 


$ git clone https://github.com/csun-metalab/meta-ui.git

#2. Next, install the necessary repo dependencies. 


$ yarn install

#3. Now with yarn, add the necessary dev dependencies.

*Please take a look at the Webpack Front-End Commands section on the Github repository to learn how to use yarn throughout development. 


$ yarn add laravel-mix
$ yarn add cross-env
$ yarn add font-awesome
$ yarn add normalize

Github Workflow & Pull Requests

It is important to understand how to properly push and pull code. Please take a moment to look through this article writen by Github: Understanding the Github Workflow.

Feature Requests

We would love to view your feature requests! Keep in mind though, that your ideas will be reviewed under the scrutiny of our developers, and thus should be well worded and detailed.

Here is a link to our Github issues page.

Product Versioning

Increment the version number according to this format:

MAJOR.MINOR.PATCH. MAJOR when you create incompatible API changes, MINOR when you add functionality that is backwards-compatible, and PATCH when you make backwards-compatible bug fixes.

Here's a link for more information on Symantic Versionings.

Issue Tracking & Task Management

We utilize Github's issue tracking system to manage improvements, bugs, and feature requests. This allows us to issue contributors to tasks. Please take a look at Github's Mastering Issues Guide.
